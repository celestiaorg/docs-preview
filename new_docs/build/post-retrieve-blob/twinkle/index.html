Using Twinkle to post and retrieve blobs on Celestia

TwinkleÂ  is a hosted API service that lets you interact with Celestiaâ€™s data availability layer without running your own node. With an API key, you can post and retrieve blobs on Celestia and pay for storage using a credit card.

Sign up and get an API key

Go to t.techÂ  and create an account, then check the email sent to you to verify your account. Once logged in, you can create an API key in the dashboard.

Twinkle offers multiple pricing tiersÂ , this tutorial will use the free tier which allows for testnet blob submissions only.

Even though the free tier is limited to testnet, you will still need to have a credit card on file to post free blobs. Your card will not be charged.

Post a blob

Blobs are chunks of raw data you post to Celestia. Each blob must be assigned a namespace, which is like a label or category that helps you group and later retrieve your data.

Namespaces are 20-hex-char labelsÂ  used to group blobs on Celestia. Use them to tag your data so you can query or prove inclusion. Namespaces make fetching and verifying your data efficient, you get the block-wide availability guarantee while working only with what matters to you.

To post a blob, use this curl command, replacing $TWINKLE_API_KEY with the actual API key you created above, and optionally replacing the namespace with your own.

curl -X POST https://t.tech/v0/blob \
  -H "Authorization: Bearer $TWINKLE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
        "namespace": "aabbccddeeff00112233",
        "data": "69206C6F76652062696720626C6F6273",
        "network": "mocha-4"
      }'
  • namespace: 20-char hex string (your label)
  • data: your blob data, hex-encodedÂ 
  • network: set to mocha-4 for testnet, defaults to mainnet if no value is specified

Retrieve your blob

To fetch the blob you posted, youâ€™ll need:

  • the namespace (base64-encodedÂ )
  • the blob commitment (also base64)
  • the block height

Use the /v0/blob endpoint. This example fetches the blob posted in the previous step:

curl "https://t.tech/v0/blob?namespace=AAAAAAAAAAAAAAAAAAAAAAAAAKq7zN3u/wARIjM=&blobCommitment=nvxg4w9nCI13RZXc9EO3T2/VBMi9XwIsEoWjx37Khgs=&height=8588106&network=mocha-4" \
  -H "Authorization: Bearer $TWINKLE_API_KEY"

Twinkle will return the blob in base64 format. Decode it to get your original data.

When posting a blob, you send both the namespace and the blob data in hexadecimal format. When retrieving, you switch to base64. This is because Twinkle converts and stores blob data in base64 internally once itâ€™s published to Celestia. Using base64 on retrieval ensures youâ€™re querying the same encoding the chain uses for commitments and proofs. In short: write in hex, read in base64.
